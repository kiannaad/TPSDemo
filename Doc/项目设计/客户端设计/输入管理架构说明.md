# 输入管理架构说明

文档类型：项目设计
关联模块：客户端输入管理
决策状态：草案

## 当前目标

当前输入管理架构的目标是把 Unity New Input System 和具体业务模块隔离开：

```text
设备输入
-> Unity InputAction
-> CGame 输入管理层
-> 输入快照 / Action 回调
-> 角色、载具、UI 等业务模块
```

输入层不直接修改角色速度、动画或战斗状态，而是提供稳定的输入语义。

## 核心结构

### PlayerInput

`PlayerInput.cs` 是 `.inputactions` 自动生成类。

它负责包装：

- `InputActionAsset`
- `Player` ActionMap
- `Vehicle` ActionMap
- `UI` ActionMap
- 自动生成的整组回调接口

这个文件不应该手动写业务逻辑，因为重新生成时会被覆盖。

### InputManager

`InputManager` 是当前输入系统的生命周期入口。

它负责：

- 创建自动生成的 `PlayerInput`。
- 创建不同输入方案的 `InputHandle`。
- 默认启用 `Player` 输入方案。
- 每帧 Tick 所有输入容器，刷新输入快照。
- 切换输入方案，例如 `Player -> Vehicle`。

当前已有输入方案：

```text
Player
Vehicle
```

`UI` 已经在枚举和自动生成类中存在，但还没有对应 `UIInputContainer`。

### InputContainer

`InputContainer` 绑定一个具体的 `InputActionMap`。

它负责：

- 启用整个 ActionMap。
- 禁用整个 ActionMap。
- 启用单个 Action。
- 禁用单个 Action。
- 注册单个 Action 的指定阶段回调。
- 移除单个 Action 的指定阶段回调。

Action 查找规则是：枚举名必须和 `.inputactions` 中的 Action 名一致。

例如：

```text
PlayerAction.Jump -> Player/Jump
VehicleAction.Brake -> Vehicle/Brake
```

### InputHandle

`InputHandle` 是对业务层暴露的门面。

它隐藏具体容器类型，对外提供：

```csharp
Enable()
Disable()
EnableAction(...)
DisableAction(...)
AddActionCallback(...)
RemoveActionCallback(...)
GetState<TState>()
AddCallbacks(...)
RemoveCallbacks(...)
```

业务模块正常情况下应该通过 `InputHandle` 使用输入系统，而不是直接访问自动生成的 `PlayerInput`。

### InputState

输入快照是每帧读取到的业务语义状态。

`PlayerInputState` 当前包含：

```text
MoveInput
LookInput
FirePressed
FireHeld
JumpPressed
SprintHeld
AimHeld
```

`VehicleInputState` 当前包含：

```text
SteerInput
ThrottleValue
BrakeHeld
ExitPressed
```

输入快照适合 locomotion、载具控制、动画参数等每帧消费的逻辑。

## 回调能力

当前有两种回调注册方式。

### 整组 ActionMap 回调

自动生成类提供整组接口，例如：

```csharp
PlayerInput.IPlayerActions
```

通过：

```csharp
AddCallbacks(...)
RemoveCallbacks(...)
```

可以一次性注册整张 `Player` ActionMap 的回调。

这种方式适合完整接管某个输入方案的模块，但不适合只监听一个 Action。

### 单 Action 回调

为了支持只监听某个 Action，当前新增了 Action 粒度 API：

```csharp
AddActionCallback(action, phase, callback)
RemoveActionCallback(action, phase, callback)
```

阶段由 `InputCallbackPhase` 表示：

```text
Started
Performed
Canceled
```

示例：

```csharp
InputHandle playerHandle = inputManager.GetHandle(InputType.Player);

playerHandle.AddActionCallback(
    PlayerAction.Jump,
    InputCallbackPhase.Performed,
    OnJump);
```

这样只会监听 `Player/Jump` 的 `performed` 阶段，不会因为 `Move`、`Look`、`Fire` 等 Action 触发。

## 禁用能力

当前输入管理支持两个粒度的禁用。

### 禁用整个输入方案

例如禁用整个 Player 输入方案：

```csharp
inputManager.GetHandle(InputType.Player).Disable();
```

适合：

- 打开菜单时禁止角色输入。
- 过场动画中禁止玩家操作。
- 切换到载具输入时关闭角色输入。

### 禁用单个 Action

例如禁用跳跃：

```csharp
inputManager.GetHandle(InputType.Player).DisableAction(PlayerAction.Jump);
```

适合：

- 硬直时禁止跳跃。
- 武器冷却时禁止开火。
- 特定状态下禁止冲刺。

需要注意：业务状态规则不一定都要通过禁用 Action 实现。比如体力不足不能冲刺，也可以在角色状态过滤层屏蔽 `SprintHeld`。

## 推荐使用方式

持续状态输入优先读输入快照：

```text
MoveInput
LookInput
FireHeld
SprintHeld
AimHeld
```

瞬时输入可以读输入快照，也可以注册单 Action 回调：

```text
JumpPressed
FirePressed
ExitPressed
```

如果逻辑属于“每帧控制”，优先使用 `GetState<TState>()`。

如果逻辑属于“某个事件发生时触发一次”，可以使用 `AddActionCallback(...)`。

## 测试与调试

自动化测试位于：

```text
Assets/Tests/EditMode/Input/InputManagerTests.cs
```

覆盖内容包括：

- 禁用单个 Action。
- 禁用整个输入方案。
- 切换 `Player / Vehicle` 输入方案。
- Player 各按键写入 `PlayerInputState`。
- Vehicle 各按键写入 `VehicleInputState`。
- 单 Action 回调注册和移除。

运行时手动调试脚本位于：

```text
Assets/Script/3C/Input/Debug/InputRuntimeLogTester.cs
Assets/Script/3C/Input/Debug/InputRuntimeToggleTester.cs
```

`InputRuntimeLogTester` 用于打印输入快照。

`InputRuntimeToggleTester` 用于测试输入方案切换、禁用当前方案、禁用 Player Jump。

## 当前缺口

当前架构还没有补齐：

- `UIInputContainer` 和 `UIInputState`。
- 玩家控制权层，例如类似 UE `PlayerController -> Pawn` 的关系。
- 输入过滤层，例如根据角色状态过滤输入快照。
- 上下文栈或优先级系统，目前只有简单的 `SwitchMap(from, to)`。

这些不影响当前输入模块的基础验证，但会影响后续多角色、UI、载具和复杂状态切换的扩展。
